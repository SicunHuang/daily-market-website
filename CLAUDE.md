## Design Conventions

These are intentional design decisions — preserve them when modifying the website:

- **Glass navbar**: The top navigation bar uses a semi-transparent frosted glass effect (`bg-nav-bg/40 backdrop-blur-2xl backdrop-saturate-150`). Do not make it fully opaque.
- **Bidirectional scroll fade**: All major sections are wrapped in `<ScrollAnimator>`. Content fades in (opacity 0→1, translateY up) as it enters the bottom 25% of the viewport, and fades out (opacity 1→0, translateY up) as it exits the top 25%. This is a continuous effect driven by scroll position, not a one-time entrance animation.
- **Sector Heatmap height**: The TradingView sector heatmap is set to 800px tall to show individual company names and logos. Do not reduce below this.
- **Color-coded market impact**: Directional arrows (↑/↓) in `marketImpact` text are colored green/red respectively via the `ColoredMarketImpact` component.
- **Also Mentioned card accents**: Cards in the "Also Mentioned in Headlines" section use a left-border accent — blue for ETFs, green for individual stocks.
- **Theme persistence**: The dark/light theme choice is saved to `localStorage` and restored on page load. Theme transitions use `transition: background-color 0.2s, color 0.2s`.
- **Mobile navbar**: Section navigation is visible on mobile as a horizontally scrollable pill bar (not hidden).
- **Focus rings**: All interactive elements must have `focus-visible:ring-2 focus-visible:ring-accent-blue` for keyboard accessibility.

## Data Schema

The daily briefing JSON (`data/latest.json`) includes these fields that power website features:

- `marketMood` (string): One-sentence market summary displayed as a banner at the top
- `upcomingEvents` (array): Economic calendar entries with `date`, `event`, `significance` ("high"/"medium"/"low")
- `tickerDescriptions` (Record<string, string>): Maps every ticker in `topStories[].tickers` to a one-sentence description
- These fields are generated by the scheduled task (`market-briefing-daily`). If adding new data-driven features, update both the `BriefingData` interface in `page.tsx` and the scheduled task SKILL.md.
